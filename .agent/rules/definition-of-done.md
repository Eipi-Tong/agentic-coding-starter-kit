# Definition of Done

A task is considered complete only when all conditions below are met.

---

# 1. Code Implementation

The requested functionality is implemented.

Code must:

- Follow project conventions
- Be readable and maintainable
- Avoid unnecessary complexity

---

# 2. Tests

Where applicable:

- Tests must be added or updated
- Existing tests must continue passing

If tests are not applicable, the agent should explain why.

---

# 3. Lint / Type / Build

The project must remain in a healthy state.

Examples:

- Lint passes
- Type checks pass
- Build succeeds

If checks cannot run, the agent must note this.

---

# 4. Documentation

Docs must be updated if the change affects:

- Setup instructions
- Public APIs
- Architecture decisions
- Developer workflows
- User-visible behavior

Docs should not be rewritten unnecessarily.

---

# 5. Assumptions

If assumptions were made:

They must be documented.

Example:

- Default pagination size
- Temporary API behavior

---

# 6. Summary

The agent must provide a final summary including:

- What was implemented
- Files changed
- Remaining risks
- Suggested next steps
